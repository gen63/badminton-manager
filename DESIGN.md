# デザインガイドライン

バドミントン練習管理アプリのUIデザイン統一ルール

---

## 1. 余白のルール

### 画面端の余白
- **最低20pxの余白を死守する**
- すべての要素は画面端から内側に配置
- 圧迫感を防ぎ、モダンな印象を与える

### 垂直方向の余白
- セクション間: `space-y-4`（16px）
- カード内の要素間: `space-y-2`（8px）
- ラベルとコンテンツ間: `mb-2`（8px）

### 水平方向の余白
- ボタン間: `gap-2`（8px）または `gap-3`（12px）
- カード内パディング: `p-4`（16px）

---

## 2. カードデザイン

### 基本スタイル
```
bg-white rounded-2xl shadow-sm p-4
```

### カードの使い方
- バラバラな情報を直接背景に置かない
- 白い角丸のボックス（カード）の中に情報を整理
- 構造がスッキリ見える

### カード内の構成
- タイトル → コンテンツ → アクション（上から下）
- 左揃えを基本とする
- 重要な情報は上に配置

---

## 3. 配色ルール

### メインカラー（3色に絞る）
| 用途 | 色 | Tailwind |
|------|-----|----------|
| メイン（アクション） | 青 | `blue-500` |
| 成功（開始・完了） | 緑 | `green-500` |
| 警告（削除・終了） | 赤/オレンジ | `red-500` / `orange-500` |

### ニュートラルカラー
| 用途 | 色 | Tailwind |
|------|-----|----------|
| 背景 | 薄いグレー | `gray-100` |
| カード背景 | 白 | `white` |
| テキスト（メイン） | 濃いグレー | `gray-800` |
| テキスト（サブ） | 中間グレー | `gray-500` / `gray-600` |
| ボーダー | 薄いグレー | `gray-200` |
| 無効状態 | 薄いグレー | `gray-400` + `opacity-50` |

### 色の使い分け
- それ以外は薄いグレーと黒で構成
- 同じ意味には同じ色を使う
- 色のトーンを統一する（彩度・明度を揃える）

---

## 4. タイポグラフィ

### フォント
- **ゴシック体で統一**（システムフォント）
- 明朝体は使用しない（他のUI要素と噛み合わない）

### フォントサイズのヒエラルキー

#### 基本原則
- タイトルは「大きくして目立たせる」のではなく「余白で際立たせる」
- サイズの差（ジャンプ率）を適切に保ち、視線が飛びすぎないようにする
- 太字であれば小さくても十分に目立つ

#### 推奨サイズスケール（iPhone基準）
| 役割 | サイズ | Tailwind | スタイル |
|------|--------|----------|----------|
| メインタイトル | 22-24px | `text-xl` / `text-2xl` | Bold |
| セクション見出し | 16-18px | `text-base` | Bold / Semibold |
| 本文・ラベル | 14-15px | `text-sm` | Regular |
| 補足・注釈 | 12px | `text-xs` | Regular / Gray |

#### 現在の実装（控えめスタイル）
| 用途 | Tailwind |
|------|----------|
| ページタイトル | `text-base font-medium` |
| セクションタイトル | `text-base font-semibold` |
| 本文 | `text-sm` |
| 補足テキスト | `text-xs` |
| ボタン | `text-xs font-medium` |

### タイトル周りの余白ルール

#### 原則
- 文字サイズではなく**余白で存在感を出す**
- すべての画面で「上端からタイトルまでの距離」を統一
- 画面遷移時のガタつきを防ぐ

#### 推奨設定
```css
/* タイトルの基本スタイル */
h1 {
  font-size: 24px;        /* 大きすぎない */
  font-weight: bold;
  text-align: left;       /* 左寄せでモダンに */
  padding: 20px 16px;     /* 余白で存在感を出す */
  margin: 0;
  font-family: sans-serif;
}
```

#### Tailwind実装例
```
text-xl font-bold text-left px-4 py-5
```

### ラージタイトルパターン（参考）

iPhoneの標準アプリに見られるパターン：

| 状態 | タイトルサイズ | 配置 |
|------|--------------|------|
| 初期状態 | 大きく表示（32px） | コンテンツエリア上部 |
| スクロール後 | 小さく表示（17px） | ナビゲーションバー内に固定 |

**メリット**: 最初に「何の画面か」を伝えつつ、操作中はコンテンツ領域を最大化

### 文字色の優先順位
1. **主要情報**: `text-gray-800`（濃い）
2. **副次情報**: `text-gray-600`
3. **補足情報**: `text-gray-400`（薄い）

## 5. コンポーネントの統一

### ボタンスタイル

#### プライマリボタン（メインアクション）
```
bg-blue-500 text-white rounded-full font-medium hover:bg-blue-600
```

#### セカンダリボタン（サブアクション）
```
bg-gray-100 text-gray-600 rounded-full font-medium hover:bg-gray-200
```

#### 危険ボタン（削除など）
```
bg-red-500 text-white rounded-full font-medium hover:bg-red-600
```

### 選択状態
- **選択中**: `bg-blue-500 text-white shadow-lg ring-2 ring-blue-300 scale-105`
- **非選択**: `bg-gray-100 text-gray-500`
- チェックマーク（✓）を表示

### 入力フィールド
```
bg-gray-50 border border-gray-200 rounded-xl px-4 py-3 
focus:ring-2 focus:ring-blue-300 focus:border-transparent
```

### アイコン
- 線の太さを統一
- サイズは `size={18}` または `size={20}` を基本
- 絵文字は控えめに使用

---

## 6. 情報の構造化

### リスト表示
- 単なるテキストの羅列を避ける
- 各項目をカード化またはボーダーで区切る
- 連番・名前・スコアなど、情報を視覚的に分離

### 構成例
```
[連番] [チームA名] vs [チームB名]
       [時刻] ([経過時間]) [スコア]
```

### アクションボタンの配置
- 右側に配置
- アイコンのみでコンパクトに
- ホバー時に色が変わる

---

## 7. レスポンシブ対応

### モバイルファースト
- 主要なターゲットはスマートフォン
- 3カラムまで横スクロールなしで表示

### iOS Safari対応
- `box-sizing: border-box` を明示
- `-webkit-appearance: none` で独自スタイル無効化
- `max-w-full` で幅のはみ出し防止

- [ ] ボタンスタイルは統一されているか
- [ ] 選択状態が明確か
- [ ] 情報の構造が視覚的に分かりやすいか
